# FaceCheck历史记录-UI与功能重构（精简版）

## 保留的有效结论
- 登录循环问题已通过“统一登录态 + 资料页角色兼容 + 移除强制跳转”解决。
- 设置页已完成 `SettingsActivity -> SettingsFragment` 迁移，并接入底部导航统一体验。
- 设置相关历史问题（主题切换、图标高亮、WebDAV交互）已有明确改造方案并落地。

## 已过期/需标注迁移的内容
- 历史方案中涉及本地人脸检测与本地推理（含 `RetinaFaceTFLiteDetector`、ML Kit 等）已不再作为主链路。
- 当前方向为服务端识别，客户端仅保留采集、上传与结果承载能力。

## 当前建议
- UI 文档后续仅记录“可见交互变更 + 页面结构变更”。
- 模型/识别引擎细节统一迁移到“架构与识别链路文档”，避免 UI 文档混入过时实现细节。
