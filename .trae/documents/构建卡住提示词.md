### 终极构建修复脚本 (PowerShell) - “用后即焚”版

**用法**: 在项目根目录的 PowerShell 终端中，直接复制并执行以下完整脚本。

```powershell
# --- Ultimate Build Fix Script (Leave No Trace) ---

# 1. 编译前：强制杀死所有 Java 进程以释放文件锁。
Write-Host "Step 1: Pre-build cleanup - Killing all Java processes..." -ForegroundColor Yellow
Get-Process -Name java,javaw -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
Write-Host "All Java processes stopped." -ForegroundColor Green

# 2. 编译前：停止所有正在运行的 Gradle 守护进程。
Write-Host "Step 2: Pre-build cleanup - Stopping Gradle daemons..." -ForegroundColor Yellow
.\gradlew.bat --stop
Write-Host "Gradle daemons stopped." -ForegroundColor Green

# 3. 编译前：强制删除所有构建缓存和中间文件。
Write-Host "Step 3: Pre-build cleanup - Wiping build directories..." -ForegroundColor Yellow
if (Test-Path '.\app\build') { Remove-Item '.\app\build' -Recurse -Force -ErrorAction SilentlyContinue }
if (Test-Path '.\.gradle') { Remove-Item '.\.gradle' -Recurse -Force -ErrorAction SilentlyContinue }
Write-Host "Build directories cleaned." -ForegroundColor Green

# 4. 编译时：使用 --no-daemon 模式启动一个全新的、干净的构建。
Write-Host "Step 4: Building - Starting clean build with --no-daemon..." -ForegroundColor Cyan
.\gradlew.bat --no-daemon :app:assembleDebug
$buildExitCode = $LASTEXITCODE

# 5. 编译后：再次强制杀死所有 Java 进程，确保环境干净，不影响 Android Studio。
Write-Host "Step 5: Post-build cleanup - Killing all Java processes again to release locks..." -ForegroundColor Yellow
Get-Process -Name java,javaw -ErrorAction SilentlyContinue | Stop-Process -Force -ErrorAction SilentlyContinue
Write-Host "All Java processes killed. Environment is clean for other IDEs." -ForegroundColor Green

# 6. 报告构建结果
if ($buildExitCode -eq 0) {
    Write-Host "Build Succeeded!" -ForegroundColor Green
} else {
    Write-Host "Build Failed! (Exit Code: $buildExitCode)" -ForegroundColor Red
}
```

### 这个脚本做了什么：

1.  **编译前彻底清理**: 在编译开始前，强制杀死所有 `java.exe` 和 `javaw.exe` 进程，停止 Gradle 守护进程，并删除 `app/build` 和 `.gradle` 目录，确保一个完全纯净的编译环境。
2.  **隔离编译**: 使用 `--no-daemon` 参数执行编译，强制 Gradle 在一个全新的、临时的、一次性的进程中运行，避免使用可能已卡死或状态异常的后台守护进程。
3.  **编译后彻底退出**: 这是最关键的一步。在编译**结束**后，脚本会**再次**强制杀死所有 Java 进程。这确保了即使 `--no-daemon` 进程有任何残留，也会被彻底清除，从而将一个完全“解锁”的、干净的环境交还给您的 Android Studio 或其他编辑器。

### 2026-04-05 补充：本项目固定构建命令（Java 17+）

```powershell
$env:JAVA_HOME='C:\Program Files\Java\jdk-17'
$env:Path="$env:JAVA_HOME\bin;$env:Path"
java -version
.\gradlew :app:assembleDebug --no-daemon
```

若提示 `Android Gradle plugin requires Java 17 to run`，说明当前 Java 版本过低，请切换到 17 或更高版本后重试。
